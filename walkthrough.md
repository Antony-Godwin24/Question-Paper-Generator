# Question Paper Generator - Walkthrough

This guide will help you run and test the Question Paper Generator application.

## Prerequisites
1. **Java 17** or higher installed.
2. **Maven** installed.
3. **MySQL** running on localhost:3306.
4. Database `QnPaper` created in MySQL.

## Running the Application

1. **Open Terminal** in the project root:
   ```bash
   cd "c:\Users\ri3ha\OneDrive\Desktop\Desktop\SEM 5\SEM 5 PRJCTS\OOAD\other\QuestionPaperGenerator"
   ```

2. **Run with Maven**:
   ```bash
   mvn spring-boot:run
   ```
   Wait for the application to start. You should see `Started QuestionPaperGeneratorApplication` in the logs.

3. **Access the App**:
   Open your browser and go to: [http://localhost:8080/index.html](http://localhost:8080/index.html)

## Testing the Flow

### 1. Registration
- Click on the **Register** tab.
- Enter a username (e.g., `teacher1`) and password.
- Select Role: **Teacher**.
- Click **Register**. You should see a success alert.

### 2. Login
- Switch to the **Login** tab.
- Enter the credentials you just created.
- Click **Login**. You will be redirected to the Dashboard.

### 3. Generate Question Paper
- In the Dashboard, go to **Generate Paper**.
- Enter Title: `Physics Mid-Term`.
- Subject: `Physics`.
- Difficulty: `Medium`.
- Questions: `5`.
- Click **Generate with AI**.
- Wait for a few seconds. The system calls Google Gemini API.
- Once done, you will see a success message and a **Download PDF** button.

### 4. Download PDF
- Click **Download PDF**.
- A PDF file named `paper_{id}.pdf` will be downloaded containing the generated questions.

### 5. View History
- Click on **My History** in the sidebar.
- You should see the paper you just generated listed there.

### 6. Admin Features (Optional)
- Register another user with Role: **Admin**.
- Login as Admin.
- You will see an extra button **All History (Admin)** in the sidebar, allowing you to view papers generated by all users.

## Troubleshooting
- **Database Connection Error**: Check if MySQL is running and the credentials in `src/main/resources/application.properties` are correct.
- **Gemini API Error**: Ensure the API Key in `application.properties` is valid.
